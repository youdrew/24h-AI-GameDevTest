# 关卡设计

## 难度等级

参数由公式自动计算（见 `discussion/difficulty-formula.md`）：

| 难度   | 关卡范围 | 图案类型数 | 每种组数 | 总瓦片数 | 层数 | 说明 |
|--------|----------|------------|----------|----------|------|------|
| 入门   | 1–9      | 3          | 1        | 9        | 1    | 教程关卡，逐步引导     |
| 简单   | 10–34    | 4–6        | 1        | 12–18    | 1–2  | 逐步增加图案类型 |
| 中等   | 35–69    | 6–9        | 2        | 36–54    | 2–3  | 每种图案出现 2 组 |
| 困难   | 70–99    | 10–12      | 3        | 90–108   | 3–4  | 大版面 + 复杂堆叠 |
| 专家   | 100+     | 13+        | 3+       | 117+     | 5+   | 极限挑战，解法稀少 |

## 公式

> 使用平滑插值替代 `floor`，消除难度跳变。详见 `discussion/difficulty-formula.md`。

```
smoothParam(N, period):
    base = floor(N / period)
    frac = (N % period) / period
    smoothed = frac * frac * (3 - 2 * frac)   // ease-in-out
    return round(base + smoothed)

patternTypes(N) = smoothParam(N, 10) + 3
setsPerType(N)  = smoothParam(N, 35) + 1
tileCount(N)    = patternTypes(N) × setsPerType(N) × 3
layers(N)       = smoothParam(N, 25) + 1
```

tileCount 始终由 patternTypes × setsPerType × 3 推导，数学上保证每种图案数量是 3 的倍数。

## 图案库

使用简单、易于区分的图形图标（emoji 或 SVG）：

- 水果：🍎🍊🍋🍇🍓🍑🍒🫐
- 动物：🐱🐶🐰🐻🦊🐼🐸🦁
- 自然：🌸🌻🌺🍀🌙⭐🌈❄️
- 物品：🎈🎯🎁🔔💎🎵🔑🧩

图案按图案类型数从中顺序选取。例如 types=3 取前 3 个（🍎🍊🍋），types=6 取前 6 个（🍎🍊🍋🍇🍓🍑），以此类推。

## 教程系统（关卡 1–9）

关卡 1–9 使用固定 3 种图案、每组 1 组、单层，同时提供引导式教学：

| 关卡 | 教学目标 | 引导方式 |
|------|----------|----------|
| 1–2  | 基本点击 + 三消 | 全屏遮罩仅露出可用瓦片，点击后自动飞入槽位；第 1 块飞入后显示"点击相同图案的瓦片"提示浮窗 |
| 3–4  | 覆盖概念 | 一块瓦片被半透明覆盖，引导提示："上层的瓦片移走后才能点击下层的" |
| 5–6  | 洗牌道具 | 首次赠送洗牌道具，弹出提示："版面太乱？试试洗牌🔄"，高亮洗牌按钮 |
| 7–8  | 撤销道具 | 首次赠送撤销道具，在槽位接近满时提示："走错了一步？点撤销↩️" |
| 9    | 提示道具 | 首次赠送提示道具，提示："找不到匹配？用提示💡" |

**引导规则**：
- 每种引导只在该功能首次出现时触发一次（通过 localStorage 标记 `tutorialSeen.{feature}`）
- 引导使用半透明遮罩 + 高亮目标元素 + 简短文字气泡（非模态弹窗）
- 玩家可随时点击任意位置关闭引导气泡
- 引导动画不暂停游戏，可在引导期间正常操作
- 教程期间（1-9 关），通关奖励只能选择**已介绍过的道具**（如第 3 关通关时只能选洗牌，因为撤销和提示尚未介绍）
- 9 关通关后显示"教程完成！准备迎接挑战"过渡动画，此后所有已解锁道具均可作为奖励选择

# 连击系统

## 机制概述

玩家在短时间内连续成功消除，会触发连击（Combo）。
连击达到特定阈值时，除视觉和音效反馈外，还会触发**被动效果**。

## 连击阶梯

| 连击数 | 视觉文本     | 被动效果 |
|--------|--------------|----------|
| x2     | Great!       | 无       |
| x3     | Excellent!   | 无       |
| x4     | Unbelievable!| 无       |
| x5+    | Combo Master!| ⚡ 闪电：自动消除 1 组同色瓦片（系统选剩余最少的图案） |

## 连击音效系统

每次连击消除的音效通过**程序化音频变换**实现——不使用独立音频文件，而是对基础消除音进行实时参数调整：

| 连击数 | 音频变换实现 | 合成参数 |
|--------|-------------|---------|
| x2     | 基础消除音 + 升高半音 | C-E-G 和弦，根音升高 1 半音（C#5-F-G#） |
| x3     | 基础消除音 + 升高 2 半音 | 根音升高 2 半音（D5-F#-A） |
| x4     | 基础消除音 + 升高 3 半音 | 根音升高 3 半音（D#5-G-A#） |
| x5+    | 闪电独立音效 | 雷鸣（低频白噪声衰减）+ 电击（高频锯齿波扫频），约 500ms |

**实现原理**：
- x2–x4：复用基础三消音效（`OscillatorNode` 播放 C-E-G 和弦），通过 `.frequency.value` 实时升调
- x5+：闪电是独立的合成音效（`AudioBufferSourceNode` 生成白噪声 + `OscillatorNode` 锯齿波），与连击升调无关
- 所有音效均可叠加：连击升调音 + 闪电音 + 视觉文本动画同时触发，互不阻塞

## 闪电被动效果

x5+ 触发闪电效果：
- **视觉**：全屏白光闪烁 + 目标瓦片放电粒子特效
- **音效**：雷鸣 + 电击（独立合成，不经过连击升调变换）
- **效果**：系统自动选择剩余数量最少（且 ≥ 3 块）的图案类型，消除该类型的 1 组（3 块）瓦片
- **与炸弹的区别**：闪电只消 1 组（3 块）且由系统自动选择图案；炸弹由玩家选择图案并消除该图案的全部瓦片

## 连击计数规则

> **2026-04-27 v4 更新（当前）**：combo **只增不减**。链断开、3 秒不点、撤销道具都
> **不会**让 combo 衰减。combo 只在累积到 10 格时**自动触发闪电**并归零，重新开始下一轮。
> 链（chainPattern / chainCount）独立维护，仅决定下一次三消能否给 combo +1，本身不再影响 combo 数值。

### 定义

- **chainPattern**：玩家最近一次点击的瓦片图案（也即"当前同图案连点链"的所属图案）。
- **chainCount**：自最近一次图案切换以来，连续点击同一图案的次数。第一次 = 1。
- **链维持**：点击同 chainPattern → chainCount += 1。
- **链切换（不再叫"中断"，因为 combo 不会被它影响）**：点击不同图案 → chainPattern = 新图案，chainCount = 1。
- **combo +1 时机**：触发三消的那次点击同时满足 chainCount ≥ 3。也就是说被消掉的那 3 块完全来自当前未切换图案的连点链。
- **combo 上限 / 重置**：combo 累积到 `COMBO_METER_MAX`（默认 10）时，**自动**触发一次闪电特效（自动消除一组同色瓦片），然后 combo = 0，开始下一轮。
- **撤销道具**：只回滚链（chainCount → 0、chainPattern → null），combo 数值不变。
- **下落瓦片消除**：不算入 chain，也不推进 combo（只有玩家主动点击的瓦片进 slot 才算）。
- **槽位满时点击**：被拒绝并播放警告音；不影响 combo，也不重置链。

### 标准示例

设槽位当前为 `[A, B, C]`（chainPattern = C，chainCount = 1，假设 combo 当前为 4）：

- ✅ **再点 `C, C`**：chainCount 从 1 → 2 → 3，第二次 C 触发三消时 chainCount = 3，**combo +1 = 5**。
- ❌ **先点 `D`，再点 `C, C`**：点 D 把 chainCount 重置为 1（图案 = D，**combo 仍是 4**），后续两次 C 让 chainCount 走到 1 → 2，三消时 chainCount = 2 < 3，**combo 不增**仍是 4。三消本身仍发生（瓦片照消）。

### combo 满格 → 自动闪电

当 combo 到达 `COMBO_METER_MAX` 时：

1. 立刻自动触发闪电（自动消除 1 组数量最少且 ≥3 块的图案）。
2. combo 归 0。
3. 链不强制重置——上一次点的图案仍然是 chainPattern，玩家可以无缝续链开始下一轮累积。

> 玩家**不需要也不能手动**点击蓄力条释放闪电——满格立刻自动放，没有"留着不放"的操作空间。

### 版本演进

| 行为 | v1 | v2 | v3 | v4（当前） |
|------|----|----|----|-----------|
| combo +1 条件 | 任何三消 | 同图案 chain 内的三消 | chain 内 + chainCount ≥ 3 | 同 v3 |
| 链切换是否清 combo | — | 是 | 是 | **否（仅清链）** |
| 3 秒超时是否清 combo | 是 | 是 | 是 | **否（已废弃超时）** |
| 满格闪电触发 | 自动（≥5 即触发） | 同 v1 | 同 v1 + 手动释放（满 10） | **自动 @ combo == 10** |
| 闪电后状态 | 不动 | 不动 | 不动 | **combo 归零，新一轮** |

## Combo Meter（连击蓄力槽）

```
[⚡⚡⚡⚡⚡⚡⚡⚡⚡⚡]  ← 10 格蓄力条
```

- 每一格 = 一次"chain 自带凑成的三消"（与 combo +1 同节拍）。
- 中间不成三消的同图案点击不点亮（只攒 chainCount）。
- 不再有衰减、不再有手动释放：满 10 格 = 自动闪电 + 清零。
- 满格时短暂高亮一帧表示"正在放电"，随后归零开始下一轮。

## 文本动画

触发 combo +1 时，按 combo 当前值显示提示语：

| combo | 文本 | 颜色 |
|-------|------|------|
| 2 | Great! | 绿 |
| 3 | Excellent! | 绿 |
| 4 | Unbelievable! | 绿 |
| ≥5 | Combo Master! | 金 |

文本以 scale 0 → 1.2 → 1.0 弹出，800ms 后淡出。combo 在 5 与 10 之间持续显示 "Combo Master!"。

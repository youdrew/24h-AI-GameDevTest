# 排行榜 UI 与数据流

## 关卡排行榜

- 每个关卡有独立的排行榜
- 显示前 20 名：排名、玩家名、星级、步数
- 玩家自己的排名始终显示在底部（如未进入前 20）

## 全球排行榜

- 按总星数排名（所有关卡星级之和）；同星数按最少总步数排序
- 显示前 50 名

## 数据提交流程

```
玩家完成关卡
  → 客户端计算星级和步数
  → 从 localStorage 读取 playerId
  → POST Supabase RPC（upsert_record）
  → 获取当前关卡排名
  → 显示关卡完成界面 + 排名位置
```

- 提交在后台静默进行，不阻塞游戏
- 网络失败时本地缓存，下次联网重试
- 排行榜可离线查看（使用最后缓存的数据）
